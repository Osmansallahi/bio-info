Quality Control Step

QC is performed on raw FASTQ files immediately after sequencing. Problems detected here — if not corrected — will propagate and corrupt every downstream analysis step.

✂️

Trimming

If QC reveals quality issues (low-quality bases, adapter contamination), reads are trimmed or filtered before alignment. Tools: Trimmomatic, Cutadapt, fastp.

🗺️

Alignment

Trimmed FASTQ reads are aligned to a reference genome (FASTA + GTF/GFF), producing SAM/BAM files that can be used for variant calling or expression quantification.

📁

File Formats

FASTQ → trimmed FASTQ → SAM/BAM (alignment) → VCF (variants). Each format carries different information and requires different tools.

02 — FASTQ Format

The FASTQ File Format

The FASTQ file is the most common raw data output from NGS sequencing platforms. It is a text-based file format that stores both the nucleotide sequences of reads and the quality score associated with each base call.

Each read = exactly 4 lines
@FORJUSP02AJWD1 ← Line 1: Read identifier (instrument, flowcell, lane, tile, coordinates)
CCGTCAATTCATTTAAGTTTTAACCTTGCGGCCGTACTCCCCAGGCGGT ← Line 2: Read sequence (DNA bases)
+ ← Line 3: "+" sign (optional: followed by sequence identifier again)
AAAAAAAAAAAAA::99@::::??@@::FFAAAAACCAA::::BB@@7A? ← Line 4: Quality scores (ASCII characters, one per base)

Quality Scores (Phred)

Each ASCII character encodes a Phred quality score Q. Q = -10 × log₁₀(P) where P = probability of base call being wrong. Q20 = 99% accuracy, Q30 = 99.9% accuracy, Q40 = 99.99% accuracy.

Encoding

Quality scores are encoded as ASCII characters. Modern Illumina uses Illumina 1.5 encoding (ASCII 64 offset). The ASCII value minus the offset gives the Phred score.

Other Formats

PacBio: bas.h5/bax.h5 → HDFView. Illumina: BCL binary base call format → bcl2fastq conversion. FASTQ ORA = binary compressed format, up to 5× smaller than .fastq.gz.

03 — FastQC Tool

The FastQC Tool

FastQC performs quality control checks on raw sequence data from high-throughput sequencing pipelines. It provides a modular set of analyses to give a quick impression of whether data has any quality problems before doing further analysis.

📥

Import

Import your data directly from a FASTQ file — no pre-processing required.

🔍

Analyse

Analyses the sequence data and quality scores provided in the FASTQ file across multiple metrics and modules.

📊

Visualise

Generates summary metrics and graphs to visually assess the quality of your data.

📄

Report

Exports results as an HTML-based report. Also supports MultiQC to aggregate multiple FastQC runs into one report.

🖥️ Command Line Usage:
Run on multiple files: $ fastqc *.fq
Get help: $ fastqc --help
Aggregate reports: $ multiqc .
FastQC Report Modules
✅ Basic Statistics

Filename, encoding, total sequences, filtered sequences, sequence length, %GC.

✅ Per Base Sequence Quality

Boxplot of quality scores across all bases at each position in the read.

✅ Per Sequence Quality Scores

Distribution of mean quality scores across all reads.

⚠️ Per Base Sequence Content

Proportion of each base (A/T/G/C) at each position across all reads.

✅ Per Base GC Content

GC content across the whole read length.

✅ Per Sequence GC Content

GC content distribution compared to a modelled normal distribution.

✅ Per Base N Content

Percentage of N base calls at each position.

✅ Sequence Length Distribution

Distribution of fragment sizes across all sequences.

✅ Sequence Duplication Levels

Degree of sequence duplication in the library.

✅ Overrepresented Sequences

Sequences that appear more than expected — may indicate contamination.

⚠️ Kmer Content

Short sequence motifs (k-mers) that are enriched at specific positions.

✅ Adapter Content

Cumulative proportion of adapter sequences found at each position.

04 — Quality Metrics

FastQC Metrics — Quality Scores

📊 Per Base Sequence Quality
Shows a Boxplot (Box-and-Whisker) of quality scores across all bases at each position in the read. Y-axis = Quality Score, X-axis = nucleotide position in read. The red central line = median. Yellow box = inter-quartile range (25–75%). Upper/lower whiskers = 10th and 90th percentile points.
⚠️ WARNING: lower quartile <10 OR median <25 for any base ❌ FAILURE: lower quartile <5 OR median <20 for any base
GOOD DATA: Most reads HIGH quality — boxes stay in the green zone throughout the read length.
BAD DATA: General degradation of quality with increasing read length (chemistry limit — normal for longer reads).
📈 Per Sequence Quality Scores
Shows if a subset of sequences are of consistently low quality. Plots the distribution of mean quality scores across all reads. Good data shows a single sharp peak at a high quality score. Bad data shows a bi-modal or complex distribution with a secondary peak at low quality.
⚠️ WARNING: most frequently observed mean quality <27 (0.2% error rate) ❌ FAILURE: most frequently observed mean quality <20 (1% error rate)
05 — Content Metrics

FastQC Metrics — Sequence Content

🔤 Per Base Sequence Content
Shows the proportion of each base (A, T, G, C) called at each position across all reads. In good data, the four lines should run parallel and relatively flat — each base should appear ~25% of the time in a random library.
⚠️ WARNING: difference between A and T, or G and C >10% at any position ❌ FAILURE: difference between A and T, or G and C >20% at any position
BAD DATA may indicate: Over-represented sequences, biased library preparation, or adapter contamination.
🔵 Per Sequence GC Content
Compares the GC content distribution across all reads to a modelled normal distribution. Good data shows a smooth normal distribution centred around the expected GC% for the species. Bad data shows sharp peaks (sequence contaminant) or broad secondary peaks (species contaminant).
⚠️ WARNING: sum of deviations from normal distribution >15% of reads ❌ FAILURE: sum of deviations from normal distribution >30% of reads
❓ Per Base N Content
Shows the percentage of base calls at each position for which an N was called (N = ambiguous base, the sequencer couldn't determine the base). In good data this should be essentially zero at all positions.
⚠️ WARNING: any position shows N content >5% ❌ FAILURE: any position shows N content >20%
06 — Other Metrics

FastQC Metrics — Duplication & Adapters

📋 Sequence Length Distribution
Distribution of fragment sizes. Some libraries produce reads of uniform length; others vary. For some sequencing platforms (e.g., PacBio, Nanopore) variable read lengths are entirely normal.
⚠️ WARNING: if all sequences are not of the same length ❌ FAILURE: if any sequences have length = 0
🔁 Sequence Duplication Level
In a diverse library most sequences will occur only once. Higher duplication = PCR amplification bias. Note: higher duplication is expected in Transcriptomics (RNA-seq) where highly expressed genes naturally produce many identical reads.
⚠️ WARNING: non-unique sequences >20% of total ❌ FAILURE: non-unique sequences >50% of total
⚠️ Overrepresented Sequences
If a single sequence is very overrepresented, it may be biologically significant (e.g., rRNA) or indicate library contamination. For small RNA libraries, overrepresentation is normal since they are not randomly fragmented.
⚠️ WARNING: any sequence represents >0.1% of total ❌ FAILURE: any sequence represents >1% of total
🔌 Adapter Content
Cumulative percentage of the proportion of reads that contain adapter sequences at each position. Adapters are short artificial sequences ligated to library fragments — they must be removed before alignment. If present in reads, the insert size was shorter than the read length.
⚠️ WARNING: any adapter sequence present in >5% of all reads ❌ FAILURE: any adapter sequence present in >10% of all reads
🔤 Kmer Content
Analyses short sequence k-mers (typically 5-7 bp) that are enriched at specific positions. Platform-dependent — libraries with random priming will nearly always show Kmer bias at the start. Usually not detrimental.
⚠️ WARNING: k-mer imbalanced with binomial p-value <0.01 ❌ FAILURE: k-mer imbalanced with binomial p-value <10⁻⁵
07 — Interpretation

Interpreting FastQC — Warnings & Action

⚠️ Important: Warnings and failures should be taken cautiously but may not be detrimental for the rest of the analysis — they might even be expected depending on the library type.

🔁

Duplication — Context Matters

Genomics: Low duplication expected (random fragmentation). Transcriptomics: Higher duplication expected for highly expressed genes — a WARNING here is likely normal and acceptable.

⚠️

Overrepresented Sequences

In small RNA libraries, sequences are not randomly fragmented, so the same sequence may naturally appear very often. This is expected and not a library quality problem.

🔤

Kmer Content

Kmer bias at the start of reads is almost always platform-specific. Libraries with random priming will nearly always show this bias — it's an artifact, not a quality problem.

When to Trim

If reads show genuine quality problems, perform sequence filtering / trimming before alignment:

Mean Quality

Remove reads below a mean quality threshold (e.g., Q20).

Read Length

Remove reads shorter than a minimum length after trimming.

Quality Trimming

Trim low-quality bases from the 3' end until a quality threshold is met.

Adapter Trimming

Detect and remove adapter sequences from reads using tools like Trimmomatic or Cutadapt.

% Bases Above Threshold

Remove reads where fewer than X% of bases are above a minimum quality score.

Adapter Reads

Remove entire reads that consist primarily of adapter sequence (insert too short).

08 — Summary

What You Should Remember

  • NGS produces FASTQ files — text files containing reads (sequences) and per-base quality scores (Phred).
  • Each FASTQ read is defined by exactly 4 lines: identifier, sequence, "+", and quality scores as ASCII characters.
  • FastQC performs QC on raw FASTQ data and exports an HTML report with 12 metric modules.
  • Per Base Sequence Quality — the most important metric; quality should stay high (green zone) across all positions.
  • Per Sequence GC Content — should follow a smooth normal distribution; deviations suggest contamination.
  • Sequence Duplication — expected to be low in genomics but higher in transcriptomics (RNA-seq).
  • Warnings and failures are context-dependent — they may be expected for certain library types (Kmer, small RNA).
  • If genuine quality problems exist, perform trimming (Trimmomatic, Cutadapt) before alignment to improve downstream results.
  • MultiQC can aggregate multiple FastQC reports into a single overview — essential for batch NGS experiments.
